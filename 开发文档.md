## 1. 项目概述

### 1.1 项目定位

**课程小管家** 是一款面向小学家长的课表管理应用，帮助家长：
- 管理多个孩子的课表
- 配置每日书包携带物品
- 接收课程和书包提醒通知
- 导出课表分享给家人

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| 课表管理 | 可视化课表展示，支持按周查看、单双周切换 |
| 书包清单 | 为每天配置书包携带物品，帮助整理书包 |
| 提醒通知 | 课前提醒和书包整理提醒，支持自定义提醒时间 |
| 课表导出 | 支持导出为图片、Excel、Word、Markdown 格式 |
| 预置课表 | 内置多套常见课表模板，一键导入使用 |
| 科目颜色 | 为不同科目设置颜色，课表更直观 |
| 时间点配置 | 自定义上下课时间，适配不同学校作息 |
| 假期日历 | 显示 2025-2028 年法定假日 |

### 1.3 已移除功能

- ❌ **OCR 导入**：已移除（v1.0.0），简化应用，专注核心课表管理体验

## 4. 核心模块说明

### 4.1 数据模型（Models）

所有数据模型都继承 `HiveObject`，使用 `@HiveType` 注解定义类型 ID。

#### Student（学生）

```dart
@HiveType(typeId: 0)
class Student extends HiveObject {
  @HiveField(0) String id;
  @HiveField(1) String name;
  @HiveField(2) String avatarColor;  // 十六进制颜色
  @HiveField(3) String grade;        // 年级
  @HiveField(4) String className;    // 班级
}
```

#### Semester（学期）

```dart
@HiveType(typeId: 1)
class Semester extends HiveObject {
  @HiveField(0) String id;
  @HiveField(1) String studentId;
  @HiveField(2) String name;         // 学期名称
  @HiveField(3) DateTime startDate;  // 开学日期
  @HiveField(4) int totalWeeks;      // 总周数
  @HiveField(5) String currentWeekType;  // A/B 单双周
}
```

#### CourseSchedule（课表）

```dart
@HiveType(typeId: 2)
class CourseSchedule extends HiveObject {
  @HiveField(0) String id;
  @HiveField(1) String studentId;
  @HiveField(2) String semesterId;
  @HiveField(3) int dayOfWeek;      // 1-5 周一到周五
  @HiveField(4) int period;         // 0=早读, 1-8=第1~8节
  @HiveField(5) String subject;     // 科目名称
  @HiveField(6) String classroom;   // 教室
  @HiveField(7) String weekType;    // A/B/ALL 单双周
}
```

#### TimeSlot（时间点）

```dart
@HiveType(typeId: 4)
class TimeSlot extends HiveObject {
  @HiveField(0) String id;
  @HiveField(1) String semesterId;
  @HiveField(2) int period;         // 节次
  @HiveField(3) String label;       // 标签（如"第一节"）
  @HiveField(4) String startTime;   // 开始时间（HH:mm）
  @HiveField(5) String endTime;     // 结束时间（HH:mm）
}
```

### 4.2 状态管理（Providers）

所有 Provider 都继承 `ChangeNotifier`，使用 `notifyListeners()` 通知 UI 更新。

| Provider | 管理的 Box | 核心功能 |
|----------|------------|---------|
| `StudentProvider` | `students` | 学生 CRUD |
| `SemesterProvider` | `semesters` | 学期 CRUD |
| `ScheduleProvider` | `schedules`, `subjectColors` | 课表 CRUD + 科目颜色 |
| `TimeSlotProvider` | `timeSlots` | 时间点 CRUD |
| `ReminderProvider` | `reminders` | 提醒 CRUD + 通知调度 |
| `BackpackProvider` | `backpackItems` | 书包物品 CRUD |
| `HomeworkProvider` | `homework` | 作业 CRUD |
| `ThemeProvider` | `settings` | 主题切换 |
| `OnboardingProvider` | `settings` | 引导流程状态 |

#### Provider 初始化模式

```dart
class ScheduleProvider extends ChangeNotifier {
  final Box<CourseSchedule> _scheduleBox = Hive.box<CourseSchedule>('schedules');
  List<CourseSchedule> _schedules = [];

  // 构造函数中从 Hive 加载数据
  ScheduleProvider() {
    _loadFromBox();
  }

  void _loadFromBox() {
    _schedules = _scheduleBox.values.toList();
    notifyListeners();
  }

  // CRUD 操作
  void addSchedule(CourseSchedule schedule) {
    _scheduleBox.put(schedule.id, schedule);
    _schedules = _scheduleBox.values.toList();
    notifyListeners();
  }
}
```

### 4.3 服务层（Services）

#### NotificationService（通知服务）

**单例模式**，负责所有本地通知的调度和取消。

**核心方法**：
- `init()` - 初始化 timezone 和通知插件
- `requestPermission()` - 请求通知权限
- `showCourseReminder()` - 立即发送课程提醒
- `showBagReminder()` - 立即发送书包提醒
- `scheduleDailyCourseReminder()` - 定时每日课程提醒
- `scheduleDailyBagReminder()` - 定时每日书包提醒
- `scheduleCustomReminder()` - 定时自定义提醒
- `cancelCustomReminder()` - 取消自定义提醒
- `rescheduleAll()` - 重新调度所有提醒（App 启动时调用）

**通知渠道**：
- `course_reminder` - 课程提醒
- `bag_reminder` - 书包提醒
- `custom_reminder` - 自定义提醒

**注意**：对于 30 分钟内的提醒，使用 `Future.delayed` 确保准时触发；对于较远的提醒，使用 `zonedSchedule`。

#### ScheduleExporter（导出服务）

支持 4 种导出格式：
1. **图片** - 渲染课表 Widget 为 PNG，保存到相册
2. **Excel** - 生成 `.xlsx` 文件
3. **Word** - 生成 `.docx` 文件（使用 `docx_dart`）
4. **Markdown** - 生成 `.md` 文件

### 5.2 Box 初始化

在 `main.dart` 中安全打开所有 Boxes：

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Hive.initFlutter();

  // 注册 TypeAdapter
  Hive.registerAdapter(StudentAdapter());
  // ... 其他 Adapter

  // 安全打开 Boxes（失败时自动重建）
  try { await _safeOpenTyped<Student>('students'); } catch (_) {}
  // ... 其他 Boxes
}
```

**安全打开策略**：
- 尝试打开 Box
- 如果失败，关闭后重新创建空 Box
- 避免数据损坏导致应用无法启动

## 7. 开发规范

### 7.1 代码规范

- **命名**：
  - 文件名：小写 + 下划线（`course_schedule.dart`）
  - 类名：大驼峰（`CourseSchedule`）
  - 变量/函数：小驼峰（`getSchedulesForStudent`）
  - 常量：全大写 + 下划线（`DEFAULT_SCHEDULE`）

- **注释**：
  - 每个 Model 类添加简要说明
  - 每个 Provider 添加核心功能说明
  - 复杂逻辑添加行内注释

- **格式化**：
  - 使用 `dart format` 格式化代码
  - 行宽限制：80 字符

### 7.2 Git 规范

**分支策略**：
- `main` - 主分支，对应已发布版本
- 本地开发直接提交到 `main`

**提交信息格式**：
```
<类型>: <简要描述>

<详细描述>（可选）
```

**类型**：
- `feat` - 新功能
- `fix` - 修复 bug
- `docs` - 文档更新
- `style` - 代码格式（不影响功能）
- `refactor` - 重构
- `test` - 测试相关

**示例**：
```
feat: 添加时间点配置功能

- 新增 TimeSlotProvider
- 新增时间点设置页面
- 课表页面支持自定义时间节点
```

### 7.3 版本号规范

**格式**：`major.minor.patch+buildNumber`

- `major` - 大版本（不兼容的改动）
- `minor` - 小版本（新功能）
- `patch` - 补丁版本（bug 修复）
- `buildNumber` - 构建号（每次构建 +1）

**示例**：
- `1.0.0+1` - 首个正式版
- `1.0.25+25` - 第 25 次构建

**版本号更新位置**：
1. `pubspec.yaml` - `version: 1.0.25+25`
2. `lib/utils/app_version.dart` - `appVersion`, `appBuildNumber`
3. `CHANGELOG.md` - 添加变更记录

**签名配置**（`android/app/build.gradle`）：
```gradle
android {
    signingConfigs {
        release {
            storeFile file('../keystore/release-key.jks')
            storePassword System.getenv('KEYSTORE_PASSWORD') ?: '[YOUR_KEYSTORE_PASSWORD]'
            keyAlias 'release'
            keyPassword System.getenv('KEY_PASSWORD') ?: '[YOUR_KEY_PASSWORD]'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

**注意**：keystore 密码等敏感信息不提交到 Git，详见 `使用指南_私有版.md`。实际密码请记录在本地私有文档中。

## 9. 常见问题排查

### 9.1 构建问题

#### 问题：Gradle 依赖下载失败

**原因**：网络问题或 Maven 仓库地址不可达

**解决方案**：
1. 使用国内镜像（已有配置）
2. 手动下载依赖到 `local-maven-repo/`
3. 检查 `android/build.gradle` 中的仓库地址

#### 问题：Build completed with error

**原因**：多种可能（依赖冲突、代码错误、资源问题）

**排查步骤**：
1. 查看完整错误日志
2. 尝试 `flutter clean && flutter pub get`
3. 检查 `flutter doctor`
4. 查看 `android/` 目录下的本地配置

### 9.2 运行时问题

#### 问题：应用启动白屏

**原因**：Hive Box 打开失败或数据损坏

**解决方案**：
1. 查看 logcat 日志：`adb logcat | grep flutter`
2. 卸载应用重新安装：`adb uninstall com.example.course_manager`
3. 检查 `_safeOpenTyped` 是否触发重建

#### 问题：通知不推送

**原因**：
- 通知权限未授予
- 定时时间已过期
- 时区数据未初始化

**排查步骤**：
1. 检查通知权限：设置 → 应用 → 通知
2. 查看 logcat 中的通知日志
3. 确认 `timezone.initializeTimeZones()` 已调用
4. 确认 `requestPermission()` 已调用并返回 true

#### 问题：课表数据丢失

**原因**：Hive Box 被重建（数据损坏）

**解决方案**：
- 目前无云端备份，数据丢失无法恢复
- 建议后续添加导出备份功能
